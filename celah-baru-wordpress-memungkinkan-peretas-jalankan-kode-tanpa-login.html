Celah Baru WordPress Memungkinkan Peretas Jalankan Kode Tanpa Login

20 Juli 2026
⏱️ Dibaca 3 menit

WordPress merilis pembaruan keamanan darurat setelah ditemukan rantai kerentanan kritis yang memungkinkan penyerang tanpa autentikasi (unauthenticated) menjalankan kode berbahaya (Remote Code Execution/RCE) di server. Kerentanan ini berada di inti (core) WordPress sehingga instalasi standar tanpa plugin tambahan pun dapat terdampak jika menggunakan versi yang rentan.

Rantai eksploitasi tersebut dikenal dengan nama wp2shell dan sebenarnya merupakan gabungan dari dua kerentanan berbeda. Kerentanan pertama adalah CVE-2026-63030, yaitu kelemahan pada mekanisme REST API Batch Endpoint yang memungkinkan permintaan diproses secara tidak semestinya. Kerentanan kedua adalah CVE-2026-60137, yaitu SQL Injection pada komponen inti WordPress. Ketika kedua bug ini digabungkan, penyerang dapat meningkatkan serangan dari SQL Injection menjadi eksekusi kode tanpa perlu memiliki akun atau melakukan login.

Kerentanan REST API Batch pertama kali ditemukan oleh Adam Kues dari Assetnote (Searchlight Cyber) dan dilaporkan melalui program HackerOne WordPress. Sementara itu, kelemahan SQL Injection dilaporkan secara terpisah oleh peneliti keamanan TF1T, dtro, dan haongo. Setelah pembaruan keamanan dirilis, detail teknis serta proof-of-concept (PoC) eksploitasi mulai dipublikasikan oleh sejumlah peneliti keamanan, termasuk repositori di GitHub.

Tidak semua versi WordPress memiliki tingkat risiko yang sama. SQL Injection telah ada sejak WordPress 6.8, sedangkan bug pada REST API Batch baru muncul mulai versi 6.9. Karena itu, hanya WordPress 6.9 dan yang lebih baru yang dapat dieksploitasi melalui rantai RCE lengkap.

Versi yang terdampak adalah sebagai berikut:

• WordPress 6.8.0 hingga 6.8.5 hanya rentan terhadap SQL Injection dan telah diperbaiki pada versi 6.8.6.

• WordPress 6.9.0 hingga 6.9.4 rentan terhadap rantai Remote Code Execution (RCE) lengkap dan telah diperbaiki pada versi 6.9.5.

• WordPress 7.0.0 hingga 7.0.1 juga rentan terhadap rantai RCE dan telah diperbaiki pada versi 7.0.2.

• WordPress 7.1 Beta 2 telah menyertakan seluruh perbaikan keamanan tersebut.

Menurut WordPress, pembaruan keamanan ini didorong melalui mekanisme pembaruan otomatis untuk banyak instalasi. Namun, administrator tetap disarankan memeriksa langsung versi WordPress yang digunakan karena situs yang menonaktifkan pembaruan otomatis belum tentu menerima patch secara otomatis.

Secara teknis, SQL Injection terjadi pada parameter author__not_in di kelas WP_Query. Penyerang dapat mengirimkan tipe data yang tidak sesuai sehingga validasi dilewati dan nilai berbahaya dimasukkan langsung ke dalam kueri basis data. Sementara itu, kelemahan pada endpoint /wp-json/batch/v1 menyebabkan kesalahan sinkronisasi pemrosesan beberapa sub-permintaan (batch request), sehingga permintaan penyerang dapat dijalankan menggunakan handler yang berbeda dan akhirnya mencapai jalur SQL Injection tanpa autentikasi.

Endpoint Batch REST API sebenarnya telah tersedia sejak WordPress 5.6 pada tahun 2020. Namun, mekanisme yang memungkinkan terjadinya kebingungan rute (route confusion) baru diperkenalkan pada WordPress 6.9 sehingga hanya versi tersebut yang terdampak oleh rantai eksploitasi penuh.

WordPress mengklasifikasikan rantai eksploitasi ini sebagai kerentanan kritis (Critical). Meski demikian, skor CVSS pada salah satu CVE hanya berada pada kategori High karena penilaian resminya lebih berfokus pada dampak akses data dibandingkan kemampuan menjalankan kode secara penuh. Oleh karena itu, para peneliti keamanan menyarankan administrator memperhatikan kedua nomor CVE tersebut secara bersamaan, bukan hanya melihat skor CVSS masing-masing.

Cloudflare juga mengungkapkan bahwa jalur RCE ini hanya berhasil dieksploitasi apabila situs tidak menggunakan persistent object cache seperti Redis atau Memcached. Namun, penggunaan Redis maupun Memcached bukanlah solusi keamanan karena hanya memengaruhi salah satu jalur eksploitasi dan tidak menghilangkan risiko SQL Injection yang mendasarinya.

Rapid7 mengonfirmasi bahwa dukungan deteksi untuk InsightVM dan Nexpose akan tersedia mulai 20 Juli 2026. Hingga pertengahan Juli 2026, kerentanan ini juga belum masuk ke dalam katalog Known Exploited Vulnerabilities (KEV) milik CISA karena belum terdapat bukti eksploitasi massal yang terverifikasi. Meskipun demikian, publikasi PoC membuat risiko serangan meningkat secara signifikan karena penyerang dapat dengan mudah mereplikasi teknik eksploitasi tersebut.

WordPress merupakan CMS paling populer di dunia dan menurut berbagai laporan industri digunakan oleh sekitar 43% dari seluruh situs web di internet. Data dari W3Techs menunjukkan WordPress mendominasi pangsa pasar CMS global sehingga setiap kerentanan pada inti WordPress berpotensi berdampak terhadap jutaan situs web di berbagai sektor.

Apabila belum dapat melakukan pembaruan, administrator disarankan menerapkan mitigasi sementara. Beberapa langkah yang direkomendasikan antara lain memblokir endpoint /wp-json/batch/v1 maupun parameter rest_route=/batch/v1 melalui Web Application Firewall (WAF), menonaktifkan akses WordPress REST API untuk pengguna anonim apabila memungkinkan, atau menggunakan plugin mitigasi sementara yang disediakan oleh peneliti hingga pembaruan dapat diterapkan. Namun, seluruh langkah tersebut hanya bersifat sementara dan bukan pengganti pembaruan resmi.

Administrator WordPress disarankan segera memperbarui situs ke versi 6.8.6, 6.9.5, atau 7.0.2 (atau versi yang lebih baru), memeriksa log akses terhadap endpoint REST API, serta memantau aktivitas mencurigakan setelah pembaruan dilakukan. Mengingat detail teknis dan proof-of-concept telah dipublikasikan secara luas, peluang terjadinya upaya eksploitasi diperkirakan akan meningkat dalam waktu dekat.
